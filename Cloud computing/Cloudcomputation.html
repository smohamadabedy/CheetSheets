STEP #2 - Choose Cloud Infrastructure

Cloud Region -> area or geographic location where physical resources are clustered, each region is isolated  from other regions and contain Zones (AZ), zones can also be isolated from each other.

Cloud Data centers: where all physical equipments are maintained.

Storage: available through local, file, object and block storage

Network: routers and switches and other common equipment + software layer + APIs, public and private cloud is defined via network configurations as well as firewalls, load balancer, CDNs and etc.

Hypervisor: install on physical hardwares and can create isolated virtual machine with different OS and resources, consist of two type:

  • type 1: install directly on the physical hardware (very common and known as Bare Metal, eg: Linux KVM )
  • type 2: install on OS on the physical hardware (common for end users known as Hosted, eg: oracle VMware)

Containers:

  • Portable and scalable and lead to agile devops
  • Used in the development part the cloud computation

Bare metal Servers: Controlled up to OS by provider, considered as dedicated and single tenant, customer can configure and customize it highly but costly, because there is no Hypervisor layer then it is best for high work-loads. These server can have GPU and be used for data science and AI.

STEP #3 - Know about the Storage

A local storage (within same server or rack) require to host files of OS (direct attached storage, none persistence)

Other storages are available through public or private networks.

Cloud storages are maintained by providers.

File Storage

(NFS)

Attach to compute node

Physical disks are attached to compute node via remote storage appliance

It can be connected to compute node via ethernet

  • Secure
  • Fault tolerant
  • Support multiple compute node

But

Speed is not guarantee to be steady

IOPS:

(Input output operation per second)

IOPS must be considered to support your application requirements

Block Storage

Attached to compute nodes

Accessible through fiber and speed of light!

  • Secure
  • Fualt tolerant
  • Low latancy

But

costly

Object Storage

(S3 API)

Attached to live servers

Accessible through APIs

Save in multiple datacenters and available for different regions

  • Less expensive
  • Infinit amount of volumn
  • Good for unstructrured data such as: backups, videos and …

But

slow

Buckets:

Files are stored in the bucket and each file has metadata (object Id, etc.)

Object storage has multiple type:

  • Standard Tiers: access frequently
  • Vault/ Archive Tiers: access once or twice a month
  • Cold Vault Tiers: access once or twice a year

CDN:

Content delivery network

Store data in distributed location and make it faster to be access by customers

  • Increase speed
  • Decreases traffic and loads
  • Increase uptime

STEP #4 - Security

VPC (virtual private cloud) consists of subnets and each one has its own security layer.

Create a subnet for hosting applications and add a public gateway to the internet.

Then create another subnet for hosting databases but private and only accessible from the first subnet.

Security must be insured by provider and User of the Cloud

User of the Cloud must implement security for administrative use, Development and customers

Data protection in the cloud must be implemented at different states

  • At rest: for stored data
  • In transit: for data transmission
  • In use: for data in memory

Monitoring increase security, decrease costs
